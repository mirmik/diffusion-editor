# Архитектурный план программы Diffusion Editor

## Назначение программы.
Редактор изображений с упором в рисование нейросетями.

## Библиотеки.
В программе применяются библиотеки экосистемы termin (в первую очередь ui), найти которые можно на один уровень иерархии выше (../)

## Модели. 
Нейросетевые модели, используемые программой лежат в директории вне программы. (директория указывается в настройках)

## Система слоёв 
Редактор поддерживает системы слоёв, последовательно собирающихся в изображение. Слои организованы в виде дерева

- Слои собираются последовательно от нижнего к верхнему, с учётом структуры дерева.
- Для ускорения сборки каждому слою ставиться в сопоставление composite кеш слоёв, лежащих под ним, с dirty флагом, маркирующим необходимость ленивого обновления. 
- Для слоя имеющего вложенные слои дополнительно сохраняется кеш вложенных слоёв (nested). Для слоя лишённого вложенных слоёв - nested==Null.
- Кеш слоёв перестраивается при необходимости сообразно логике - чем меньше делаем, тем лучше.
- Пример. В скобках указаны слои, участвующие в composite-кэше (previous) и (nested) кеше в порядке отрисовки
А      (F) (B,E,D,C)
|-C    (B) (E,D)
|.|-D  (E) ()
|.\-E  () ()
\-B    () ()
F

Пояснение. Формула кэша для слоя строится prefix = (previous) + (nested). Отсюда ((B)+(E,D)) = (B,E,D). 
Полное изображение тогда вычисляется как
composite(A) = prefix(A) + A = previous(A) + nested(A) + A
() - соответствует прозрачному (0,0,0,0) изображению (для экономии памяти допустимо не хранить).

Аттрибут "скрыт" и прозрачность учитываются в composite-кэше через аргумент `opaque`. `opaque` распространяет своё влияние на собственные данные слоя и его nested.

## Типы слоёв.
Основные слои программы: 
- Paint : поддерживает инструменты рисования, типа кистей
- Diffusion : Содержит маску и настройки для применения модели, подобные Stable Diffusion

Экспериментальный:

## Логика Diffusion слоя.
Основные параметры Diffusion слоя соответствуют упрощённому SD. Изображение для перерисовки берётся из кеша нижележащего слоя 

## Операции над слоями и аттрибуты слоёв.
Слои должны поддерживать 
- "Дублирование": по активации создаётся копия слоя рядом с копируемым (на том же уровне иерархии)
- "Слияние": добавляет текущий слой в предшествующий Paint-слой (Если предшествующий слой не Paint - отдаёт ошибку).  

Также у слоя есть
- Прозрачность: (в диапазоне 0..1) при отрисовки изображения или кеша верхних слоёв альфа канала умножается на атрибут прозрачности
- Скрыт: слой можно скрыть, что исключает его их отрисовки и участия в кеше.

## Движки (Engines).
Движки — обёртки над нейросетевыми моделями, выполняющие инференс в фоновом потоке.
Все движки разделяют общий паттерн: submit запускает daemon-поток, poll() вызывается каждый кадр и забирает результат (destructive read). Модели загружаются лениво.

- DiffusionEngine : SDXL через `diffusers`. Режимы txt2img/img2img/inpaint, переключаемые без перезагрузки весов. Поддержка IP-Adapter.
- LamaEngine : `simple-lama-inpainting`. Удаление объектов по маске.
- InstructEngine : InstructPix2Pix. Редактирование по текстовой инструкции.
- SegmentationEngine : `rembg` (u2net). Выделение фона/переднего плана в маску.

## Формат файла проекта (.deproj).
ZIP-архив, текущая версия формата: 3. Содержит `manifest.json` и директорию `layers/` с PNG-файлами.

Имя файла слоя образуется из его пути в дереве заменой `/` на `_`: слой `1/0/3` → `layers/1_0_3_image.png`. Для слоёв с маской/источником добавляются `_mask.png` и `_source.png`.

manifest.json хранит размер холста, путь к активному слою и рекурсивное дерево слоёв с их типами, аттрибутами и ссылками на файлы изображений.

## Undo/Redo стек. 
Undo/Redo программы аля gimp и покрывает операции рисования, перегенирации на diffusion слои, работы со слоями, редактирования настроек. 
В идеале, всё, что пользователь может сделать - должно быть покрыто undoredo.
